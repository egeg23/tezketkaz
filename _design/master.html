Как читать этот файл

01Структура HTML каждого экрана

Каждый экран — это .screen-wrap > .phone > .screen.SCREEN_NAME. Все классы префиксованы по экрану (.splash-mark, .tracking-sheet, .pe-hero) — CSS не конфликтует и каждый экран можно конвертировать в отдельный Flutter widget изолированно. Каркас (phone, status-bar, notch) используется только для рендера мокапа — в реальном Flutter-приложении его не нужно повторять.

02Flutter-маппинг паттернов

  • backdrop-filter: blur(20px) → BackdropFilter(filter: ImageFilter.blur(...)) внутри ClipRRect
  • radial-gradient(...) → RadialGradient внутри BoxDecoration
  • box-shadow: 0 0 24px rgba(6,193,103,0.4) → BoxShadow(blurRadius: 24, color: lime.withOpacity(0.4), spreadRadius: 0)
  • Lime pulse-кольца (status-pulse, courier-pin) → AnimationController + Transform.scale + AnimatedOpacity в цикле 2.4s
  • SVG-карта (tracking / active-order) → flutter_map + tezketkaz_dark_map_style.json (для Mapbox SDK) или Stadia "Alidade Smooth Dark" как fallback raster
  • Маршрут на карте → PolylineLayer с dashArray: [2, 10] + анимация offset через AnimationController
  • Курьер-пин с двойным пульсом → стек: внешний пульсар, средний пульсар, ядро (lime + white border 2.5px)
  • Все радиусы фиксированные: 10/14/16/18/20/22/24/100. Используй BorderRadius.circular(N)
  • Stagger fade-up на content children — AnimatedList или FadeTransition с delays 50–300ms

03Шрифты и брендинг

Используй google_fonts package. Playfair Display italic 500 — для эмоциональных headings с лаймовым курсивом в 1–2 словах (паттерн "Xush kelibsiz" / "Yangi do'konlar"). Inter — UI, body, кнопки, лейблы. JetBrains Mono — цены, числа, коды промокодов, телефоны, ID тикетов, метаданные. Брендмарк "tz" (lime square 12px-radius с молнией-кружком в верхнем правом углу) — везде, где нужен logo: splash, login header, иконка приложения.

04Локализация и валюта

Все строки — на узбекском (Ochiq, Yetkazib berildi, Buyurtmalarim, Mijoz, Kuryer, и т.д.). Валюта — sum (so'm), отображается с пробелом перед единицей: 140 000 so'm. Тысячные разряды — узким пробелом (NumberFormat('# ###') в Flutter). Для миллионов сум использовать суффикс M или ml: 1.8M so'm, 3.2 ml. Телефоны — формат +998 90 ••• 45 67. Время — формат '19:42' или '2 min oldin'.

05UZ-специфика

Платёжные провайдеры — Click (asosiy/основной), Payme, Visa, Naqd pul (наличные), Apple Pay. Адреса в формате "Chilonzor 19-mavze, 28-uy". Подъезд / этаж / домофон — отдельные поля. Зона доставки — радиус в км. Курьерское ТС обычно мото (Honda PCX как референс), номер ТС формата 01 A 234 BC. Курьер-портреты — мужские (cultural fit).

★Приоритеты сборки (3 milestones)

  1. P0 (MVP, ~10 экранов): Auth (Splash + Onboarding + Login + Name) + Home + Shop Detail + Product Detail + Cart + Order Success + Tracking. Минимальный flow до получения первого заказа.
  2. P1 (~10 экранов): Profile + Address Picker + Payment Methods + Orders history. Courier role целиком (Home + Active + Earnings + Performance + Profile). Role Switcher.
  3. P2 (~14 экранов): Catalog, Notifications, Address Book, Promo, Loyalty, Subscription (TZ+), Support (3 экрана), Shop Dashboard, Product Editor, Products Import, Shop Settings, Courier Verification.

07API контракты

Бэкенд — Node.js + Prisma + Socket.IO. Real-time события через Socket.IO для трекинга курьера и входящих заказов в shop dashboard. Авторизация — phone + SMS OTP, JWT. Множественные роли на одном аккаунте (юзер может быть одновременно mijoz и kuryer). Хранение состояния заказа — конечный автомат: draft → submitted → accepted → preparing → ready → picked_up → in_delivery → delivered с боковыми ветками cancelled и refunded.
