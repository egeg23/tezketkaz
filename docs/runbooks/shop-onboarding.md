# Подключение магазина (Shop Onboarding) — Phase 13.3.6

Этот плейбук покрывает полный процесс подключения нового магазина к TezKetKaz
во время пилотного запуска. Целевая аудитория: операторы (BD / онбординг),
которые выезжают к первым 5–10 магазинам.

> Цель пилота: запустить **5 магазинов** в первой неделе, **10** к концу
> второй недели. Каждое подключение должно занимать одну выездную встречу
> (≤ 60 минут) плюс ≤ 2 часа «домашней работы» оператора на оформление.

---

## 1. Подготовка к встрече

### Что взять с собой

- **Ноутбук** с открытыми вкладками:
  - admin-next (`https://admin.tezketkaz.uz/login`) — создание магазина
  - vendor-next (`https://vendor.tezketkaz.uz/onboarding`) — резервный путь
  - Notion-папка «Договоры» — шаблон скользящего договора (`contract-tpl.pdf`)
  - Этот плейбук
- **Демо-телефон** (Android или iPhone) с установленным buyer-приложением
  и тестовым аккаунтом `+998 99 000 00 01 / OTP 1234` (мок-режим)
- **Образец договора** (PDF + 2 распечатанные копии для подписи)
- **Печать TezKetKaz** (если у вас есть полномочия подписания)
- **QR-код** на canvas-стенд для меню (для ресторанов) — печатается заранее
  по `https://admin.tezketkaz.uz/qr/<shop_id>`
- Заряженный пауэр-банк — встречи часто проходят без розетки

### За день до встречи

1. Подтвердите время визита по WhatsApp / Telegram (магазины часто переносят).
2. Проверьте, что shop_admin пользователь у вас есть в админке (если будете
   логиниться от имени магазина для демо).
3. Если ресторан — попросите выслать **меню в PDF/Excel** заранее.

---

## 2. Встреча — 30-минутная программа

### Минута 0–5: установление контакта

- Представьтесь, объясните, что TezKetKaz — это сервис экспресс-доставки,
  ориентированный на узбекский рынок (UZS, СОЛИК-чеки, локальные платёжки).
- Сравнение: Wolt / Yandex.Eda, но **без комиссии на первые 30 дней пилота**.

### Минута 5–15: демо приложения

Покажите на демо-телефоне:

1. Карточка магазина (открыть список, кликнуть на одну из карточек).
2. Корзина → оформление → выбор способа оплаты (Click / Payme).
3. Экран курьера: трекинг доставки в реальном времени.
4. Чек: после доставки покупатель может **скачать PDF-чек** (Phase 13.3.3),
   на чеке указаны ИНН магазина и фискальный QR Soliq.uz.

### Минута 15–25: показ vendor-next dashboard

Откройте `https://vendor.tezketkaz.uz` и покажите:

- **Список заказов** — фильтры «Новый / В работе / Готов»
- **Кнопку «Принять»** и «Готов к выдаче» — типичный workflow смены статуса
- **Аналитика** — выручка за 7 / 30 дней, топ-товары
- **Импорт товаров** — XLSX по шаблону (см. ниже)
- **Расписание работы** — задаём open/close на каждый день недели

### Минута 25–30: подписание + сбор данных

Если владелец согласен — переходите к разделу 3.

## 3. Юридическое оформление

### Подписание скользящего договора

> **Внимание:** все условия и тексты согласовывает юридический отдел.
> Этот плейбук содержит только список полей, которые нужно собрать.
> Шаблон договора — `legal/contract-tpl-uz-v1.pdf` в Notion.

Подпишите 2 экземпляра — один остаётся у магазина, один везёте в офис.

### Собрать данные (фотографируйте сразу на телефон в чат «Onboarding-shop-XX»)

| Поле                        | Где взять                                          |
|----------------------------|-----------------------------------------------------|
| Название (рус / уз / англ) | Со слов владельца                                   |
| Юр. лицо (ИП / ООО)        | Из свидетельства о регистрации                      |
| **СТИР/ИНН (9 цифр)**      | Свидетельство o регистрации налогоплательщика      |
| НДС (если плательщик)      | НДС-свидетельство (опционально)                     |
| Расчётный счёт             | Реквизиты банка (для еженедельных выплат)           |
| Телефон владельца          | Личный номер                                        |
| Телефон менеджера (-ов)    | До 3 номеров — добавляются как ShopMember-ы         |
| Адрес магазина             | С точкой на карте (Yandex Maps geolocation)         |
| Часы работы                | По дням недели                                      |
| Зона доставки (км)         | Радиус от магазина в км или полигон                 |

## 4. Создание в системе

### Путь A: через admin-next (если вы — оператор TezKetKaz)

1. `https://admin.tezketkaz.uz/shops/new`
2. Заполните все поля из таблицы выше.
3. **СТИР/ИНН** идёт в поле `soliqInn`. Без этого фискальные чеки не выпускаются.
4. `soliqEnabled = false` пока магазин не отработает первые 3 дня без проблем.
   После этого — включаем (см. Phase 13.3.9 / soliq-fiscal-setup.md).
5. Сохраните — получите `shopId`.

### Путь B: vendor-next signup (если владелец предпочитает сам)

1. Откройте `https://vendor.tezketkaz.uz/signup` на их ноутбуке.
2. Введите телефон владельца → OTP.
3. Заполните форму магазина шаг за шагом.
4. На стороне backend это вызовет `POST /api/shops` с auto-membership owner.

### Добавление менеджеров (ShopMember)

Открыть `admin-next → Shop → Members → Add`. Каждый менеджер получает SMS
с приглашением. По умолчанию роль `manager` (видит заказы, может менять статус,
не может удалять магазин).

## 5. Загрузка первой партии товаров

Магазины загружают каталог через XLSX-импорт. Шаблон расположен в репо:
`fastlane/metadata/shop-import-template.xlsx` (бинарь не коммитим;
оператор скачивает с `https://admin.tezketkaz.uz/import-template.xlsx`).

### Ожидаемые столбцы (порядок важен)

| Колонка        | Тип    | Обязательно | Описание                                   |
|---------------|--------|-------------|--------------------------------------------|
| `name_ru`     | строка | да          | Название на русском                        |
| `name_uz`     | строка | да          | Название на узбекском (латиница)           |
| `name_en`     | строка | нет         | Английское название (для туристов)         |
| `description_ru` | текст | нет       | Описание на русском                        |
| `price_uzs`   | число  | да          | Цена в сумах (целое число, без пробелов)  |
| `vat_percent` | число  | нет         | НДС в процентах (0 / 12 / 15); по умолчанию 12 |
| `category`    | строка | да          | Slug категории: `food`, `produce`, `dairy`, `bakery`, `drinks`, `meat`, `grocery`, `pharmacy`, `electronics` |
| `image_url`   | URL    | нет         | Прямая ссылка на JPG/PNG (≤ 2 MB)         |

### Процесс импорта

1. В vendor-next: `Products → Import → загрузить XLSX`.
2. Backend парсит, валидирует, показывает превью.
3. Если есть ошибки в строках — operator чинит вручную в Excel и заливает заново.
4. После подтверждения — товары создаются, оператор проверяет визуально.

### Загрузка фотографий

Если у магазина нет ссылок на изображения — оператор фотографирует **топ-30
позиций** прямо на телефон (вертикальная ориентация, белый фон в идеале) и
заливает в vendor-next через карточку товара.

## 6. Проверка работоспособности (тестовый заказ)

После создания магазина и загрузки минимум 10 товаров — сделайте **реальный
тестовый заказ** с демо-аккаунта оператора:

1. Оператор оформляет заказ в buyer-приложении на ваш собственный адрес.
2. Магазин видит заказ в vendor-next, нажимает «Принять» → «Готов».
3. Назначенный курьер (можно тестовый) забирает и доставляет.
4. Чек скачивается из приложения покупателя (Phase 13.3.3).

**Если что-то не работает — НЕ запускайте магазин в продакшен.** Лучше потерять
один день, чем дать клиентам плохой первый опыт.

## 7. Софт-лонч чеклист (перед публикацией в каталоге)

- [ ] Часы работы заданы на все 7 дней
- [ ] Минимум 20 товаров с фото и ценами
- [ ] Зона доставки настроена (полигон или радиус)
- [ ] Минимальная сумма заказа задана (обычно 30 000 UZS)
- [ ] Расчётный счёт верифицирован (тестовая выплата 1000 UZS прошла)
- [ ] Менеджер протестировал кнопку «Принять» хотя бы раз
- [ ] У владельца есть номер дежурного оператора TezKetKaz в Telegram
- [ ] **soliqEnabled** остаётся выключенным первые 3 дня (для отладки)

## 8. Первая неделя — handholding

День 1 (день запуска):
- Утренний звонок в 09:00: убедиться, что магазин онлайн, часы работы корректны.
- Вечерний звонок в 21:00: сколько заказов? Были ли отказы? Жалобы?

Дни 2–3:
- Ежедневный 5-минутный созвон.
- Включить `soliqEnabled = true` после дня 3, если не было критичных багов.

Дни 4–7:
- 1 созвон в день, можно в чате.
- Если магазин не получил ни одного заказа за 48 часов — оператор разбирается:
  плохие фото? Высокие цены? Узкая зона доставки? — и предлагает правки.

## 9. Частые ошибки и подводные камни

| Проблема                                | Решение                                                    |
|-----------------------------------------|------------------------------------------------------------|
| Магазин не видит звук уведомления       | В vendor-next: Settings → включить звук + разрешения PWA   |
| Курьер не приехал — магазин злится       | Дежурный оператор переназначает курьера вручную в admin    |
| Цены в Excel загрузились с десятичной точкой | XLSX-парсер ругается — оператор пересохраняет файл как «Number» без копеек |
| Владелец требует наличный приём денег   | Включить `cash` в принимаемых способах оплаты (флаг shop)  |
| ИНН неверный — Soliq возвращает 4xx      | Проверить, что 9 цифр (юр.лицо) или 14 цифр (ИП). См. `soliq-fiscal-setup.md` |
| Магазин закрывается раньше — не отключился | Включить «Закрыто» в vendor-next → Расписание → переключатель |
| Не хватает категории                    | Категории добавляет только admin — пинг в `#onboarding` в Slack |
