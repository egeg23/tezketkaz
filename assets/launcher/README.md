# App icon and splash screen assets

This folder will hold the source artwork (1024Г—1024 PNG + adaptive
foreground/background PNGs) that the build pipeline uses to generate
platform icon sets. Until the design team delivers final assets, the app
ships with the default Flutter launcher icons; the spec below tells the
designer what to produce.

## Brand palette

The primary CTA / logo accent colour is `AppColors.primary` =
`#14A44D` (saturated green). Secondary tints:

- `primaryDark`  `#0E8B40` вЂ” pressed state, darker logo backdrops.
- `primaryLight` `#EAF8F0` вЂ” splash background, soft tint cards.
- `primarySoft`  `#D3F1DD` вЂ” large fills inside dark mode previews.

Recommended icon composition: a flat `#FFFFFF` "TK" wordmark or the
ketchup-leaf glyph centred on a `#14A44D` background tile with a 12% safe
margin (Android adaptive masks crop aggressively).

## Required source files

Place the following inside this folder and the launcher generator picks
them up automatically:

```

  ic_launcher.png                 # 1024Г—1024  RGBA  (master icon)
  ic_launcher_foreground.png      # 1024Г—1024  RGBA  (Android adaptive fg)
  ic_launcher_background.png      # 1024Г—1024  RGBA  (Android adaptive bg)
  ios_marketing_icon.png          # 1024Г—1024  RGB   (App Store listing)
  splash_logo.png                 # 512Г—512    RGBA  (centred splash mark)
```

## Splash screen

Use a centred logo on `AppColors.primaryLight` (`#EAF8F0`) for light mode
and `#0E1318` (matches dark-mode background) for dark mode. The splash is
configured in `android/app/src/main/res/drawable*/launch_background.xml`
and `ios/Runner/Base.lproj/LaunchScreen.storyboard`; both should be
regenerated by `flutter_native_splash` (recommended once the design lands).

## Regenerate icons + splash

```bash
# Add to dev_dependencies once design assets land:
#   flutter_launcher_icons: ^0.13.1
#   flutter_native_splash:  ^2.4.0

# Then:
flutter pub get
flutter pub run flutter_launcher_icons
flutter pub run flutter_native_splash:create
```

Both packages read configuration from `pubspec.yaml` вЂ” keep their config
blocks pointing at the files in this folder.
